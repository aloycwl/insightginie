Introduction to Subfeed AI Skill
--------------------------------

Subfeed AI is a comprehensive skill designed to provide AI entities with a full agentic backend for OpenClaw projects. This skill enables autonomous building capabilities where your IDE agent can self-register and start working immediately, with human involvement being optional. The skill is categorized under developer-tools and requires specific environment variables to function properly.

Core Functionality and Setup
----------------------------

The Subfeed AI skill operates through a straightforward flow that begins with checking for existing credentials in your environment variables. It looks for either a SUBFEED\_API\_KEY (starting with sf\_live\_\*) or a SUBFEED\_AGENT\_TOKEN (starting with sf\_agent\_\*). If these credentials are found, the skill proceeds directly to the entity creation phase. If not found, it guides users through a registration process to create their agent identity.

### Registration Process The registration process involves creating an agent identity through a POST request to the Subfeed API. Users provide a name and description for their agent, which returns an agentToken and agentId. These credentials are essential for all subsequent API interactions and serve as the authentication mechanism for the agent’s activities. Entity Creation and Management Once credentials are established, the skill facilitates the creation of AI entities. Users can select from available models, with the skill providing a fallback option of meta-llama/llama-3.3-70b-instruct if preferred models are unavailable. Each entity is created with a specified name and system prompt, allowing for customization of the AI’s behavior and capabilities. Testing and Validation After entity creation, the skill includes a testing mechanism where users can send a simple “Hello” message to verify that the entity is functioning correctly. This immediate feedback loop ensures that the setup process is complete and that the AI entity is ready for more complex tasks. Human Collaboration and Onboarding The Subfeed AI skill emphasizes optional human collaboration. When human involvement is desired, the skill provides a structured onboarding process. It prompts the user to specify what email should be used for the human’s Subfeed account, then creates the account and provides a setup URL. This URL allows the human to set their password and access their dashboard, where they can view everything the AI has built. Account Creation and Access The human account creation process generates both an API key and an agent token, providing the human with full account access through their dashboard. This dual-access system ensures that humans can monitor, control, and collaborate with the AI entities as needed. Advanced Capabilities and Extensions Beyond basic entity creation, the Subfeed AI skill offers numerous extensions to enhance AI capabilities. These include RAG (Retrieval Augmented Generation) for creating knowledge bases, various addons for enabling features like web search, code execution, and image generation, MCP (Model Context Protocol) for connecting to external tools like GitHub and Slack, and webhook support for accepting external service payloads. RAG Implementation The RAG system allows users to create knowledge bases that can be attached to entities, providing them with access to curated information. This feature is particularly useful for domain-specific applications where the AI needs access to proprietary or specialized knowledge. API Reference and Authentication The skill provides comprehensive API documentation covering all available endpoints. Authentication is handled through Bearer tokens, with both API keys and agent tokens functioning identically. The skill clearly distinguishes between human API keys (sf\_live\_\*) which provide full account access, and agent tokens (sf\_agent\_\*) which are scoped to the registering agent. Security Considerations Security is a primary concern, with all API calls using HTTPS encryption. Agent tokens are scoped to prevent unauthorized access, while human API keys provide full account control. The skill emphasizes that no binaries are required and all operations use curl to interact with the REST API. Additionally, RAG content is stored encrypted and only accessible by the owning account. Privacy and Trust Statement The Subfeed AI skill includes a clear trust statement, informing users that all API requests and entity data are sent to Subfeed Cloud servers. Users are advised to only install the skill if they trust Subfeed with the data being sent. The skill provides transparency about data handling and storage practices. Quick Reference and Error Handling The skill includes a comprehensive quick reference section covering base URLs, entity management operations, chat functionality, RAG operations, addon management, and model listings. Error handling is well-documented, with specific HTTP status codes explained (400 for invalid requests, 401 for missing authentication, 403 for forbidden access, 404 for not found, 409 for resource conflicts, and 429 for rate limiting). Public Endpoints Several endpoints are available without authentication, including agent registration, public entity invocation, webhook reception, and model listings. This design allows for flexible integration scenarios while maintaining security for sensitive operations. Conclusion Subfeed AI represents a powerful tool for developers working with OpenClaw projects, providing a complete agentic backend that enables autonomous AI operation with optional human collaboration. Its comprehensive feature set, clear documentation, and emphasis on security and privacy make it a valuable addition to any AI development workflow.
