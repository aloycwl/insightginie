<h2>What Is the Excalidraw Diagram Generator Skill?</h2>
<p>The Excalidraw Diagram Generator is a powerful skill that transforms natural language descriptions into professional Excalidraw diagrams. Whether you need a flowchart, mind map, system architecture, or data flow diagram, this tool creates ready-to-use <code>.excalidraw</code> files that can be opened directly in Excalidraw or used with the Excalidraw VS Code extension.</p>

<h3>Supported Diagram Types</h3>
<ul>
<li><strong>Flowcharts</strong> &#8211; Sequential processes, workflows, decision trees</li>
<li><strong>Relationship Diagrams</strong> &#8211; Entity relationships, system components, dependencies</li>
<li><strong>Mind Maps</strong> &#8211; Concept hierarchies, brainstorming results, topic organization</li>

<h2>How It Works: Step-by-Step Process</h2>
<h3>Step 1: Understand the Request</h3>
<p>The skill analyzes your description to identify:</p>
<ul>
<li>Diagram type (flowchart, relationship, mind map, etc.)</li>
<li>Key elements (entities, steps, concepts)</li>
<li>Relationships (flow, connections, hierarchy)</li>
<li>Complexity (number of elements)</li>
</ul>
<h3>Step 2: Choose the Appropriate Diagram Type</h3>
<p>Based on user intent, the skill selects the optimal diagram format. For example, &#8220;workflow&#8221; suggests a flowchart, while &#8220;mind map&#8221; indicates a concept hierarchy.</p>
<h3>Step 3: Extract Structured Information</h3>
<p>The skill parses your description into structured elements:</p>
<ul>
<li><strong>Flowcharts</strong>: Sequential steps with decision points</li>
<li><strong>Relationship Diagrams</strong>: Entities and their connections</li>
<li><strong>Mind Maps</strong>: Central topic with main branches and sub-topics</li>
<li><strong>Data Flow Diagrams</strong>: Data sources, processes, stores, and flows</li>
<li><strong>Business Flow</strong>: Actors, process lanes, and activities</li>
<li><strong>Class Diagrams</strong>: Classes, attributes, methods, and relationships</li>
<li><strong>Sequence Diagrams</strong>: Objects, lifelines, messages, and activation boxes</li>
<li><strong>ER Diagrams</strong>: Entities, attributes, keys, and relationships</li>
</ul>
<h3>Step 4: Generate the Excalidraw JSON</h3>
<p>The skill creates a complete <code>.excalidraw</code> file with:</p>
<ul>
<li>Properly positioned elements (rectangles, ellipses, diamonds, arrows, text)</li>
<li>Consistent styling with Excalifont for all text elements</li>
<li>Appropriate spacing and layout for readability</li>
<li>Color-coded elements for visual hierarchy</li>
</ul>
<h3>Step 5: Format and Output</h3>
<p>The final output is a structured JSON file containing:</p>
<ul>
<li>All diagram elements with coordinates and styling</li>
<li>App state settings for optimal viewing</li>
<li>Ready-to-use format for Excalidraw</li>
</ul>
<h3>Step 6: Save and Provide Instructions</h3>
<p>The skill saves the file with a descriptive name and provides clear instructions for opening it in Excalidraw.</p>

<h3>Layout Tips</h3>
<ul>
<li>Start positions: Center important elements, use consistent spacing</li>
<li>Spacing: 200-300px horizontal gap, 100-150px vertical gap</li>
<li>Colors: Light blue for primary, light green for secondary, yellow for important</li>
<li>Text sizing: 16-24px for readability</li>
<li>Font: Always use fontFamily: 5 (Excalifont) for all text</li>
<li>Arrow style: Straight for simple flows, curved for complex relationships</li>
</ul>
<h3>Complexity Management</h3>
<p>If a request contains too many elements, the skill may suggest breaking it into multiple diagrams or focusing on the most important components.</p>
