<h2>Understanding Credex Protocol Integration</h2>
<p>The Credex Protocol skill focuses on providing AI agents with access to unsecured credit lines, allowing them to leverage <a href="https://www.circle.com/what-we-do/usd-coin-usdc/">USDC</a> for various tasks while using their reputation as collateral. Built specifically for the Arc Network (currently on testnet), this system enables seamless financial manueverability for machines.</p>
<h2>Core Functionalities</h2>
<ol>
<li><strong>Borrowing Against Reputation</strong><br />
AI agents can apply for credit by borrowing USDC based on their proven reputation. The protocol calculates creditworthiness and establishes a credit limit dynamically based on historical performance indicators. For example,{\&#8221;creditLimit\&#8221;:\&#8221;100.000000\&#8221;} for agents with strong track records.</li>
<li><strong>Debt Management</strong><br />
The system supports debt repayment functionality, where agents can repay their borrowed amounts plus accrued interest. This action allows for credit limit expansion by 10% with satisfactory repayments. Example response for repayment{\&#8221;newCreditLimit\&#8221;:\&#8221;110.000000\&#8221;}</li>
<li><strong>Liquidity Provisioning</strong><br />
Agents can act as liquidity providers by depositing USDC into a shared pool. When assets in the pool are loaned out, providers earn a percentage of the interest generated by borrowers. The share price reflects the underlying asset value based on{\&#8221;totalAssets\&#8221;:\&#8221;1000.000000\&#8221;,\&#8221;totalShares\&#8221;:\&#8221;950.000000\&#8221;}</li>
<li><strong>Cross-Chain Functionality</strong><br />
Agents can transfer USDC between Arc Testnet and Base Sepolia via integration with Circle Bridge. This feature enhances pool utilization and agent liquidity management. Example parameter usage: bridge 10.0 arc base</li>
</ol>
<h2>Technical Requirements</h2>
<p>To leverage the Credex Protocol Skill, users must provide the following environment variables:</p>
<ul>
<li><code>WALLET_PRIVATE_KEY</code> &#8211; Private key for signing blockchain transactions</li>
<li><code>RPC_URL</code> &#8211; Arc Network connection endpoint, with the default value pointing to <code>https://rpc.testnet.arc.network</code></li>
<li><code>CREDEX_POOL_ADDRESS</code> &#8211; Address of the CredexPool contract (configurable)</li>
<li><code>CREDEX_AGENT_URL</code> &#8211; Credex agent server URL (optional)</li>
</ul>

<li>Check credit status:<br />

Key details in response:<br />

<h2>Protocol Mechanics</h2>
<ul>
<li><strong>Interest Accrual</strong>: 0.1% charged per minute in testnet (actual network parameters may vary). This is calculated and added to the principal using: <code>debt = principal + accrued_interest</code></li>
<li><strong>Credit Limit Growth</strong>: Each successful repayment increases the agent&#8217;s credit limit by 10%, capped at 10,000 USDC. This compounding effect can be visualized as: <code>currentLimit × 1.10 = newLimit</code></li>
<li><strong>Liquidity Provider Logic</strong>: When agents deposit funds, they receive LP shares proportional to the value of the <code>totalAssets</code> (which include <code>liquidity + outstandingDebt</code>). The <code>sharePrice</code> is recalculated with each pool update.</li>
</ul>
<h2>Security Considerations</h2>
<p>Users should be aware of the following when integrating with Credex Protocol:</p>
<ul>
<li><strong>Private Key Security</strong>: Always securely store the <code>WALLET_PRIVATE_KEY</code> and never expose it in shared environments. The protocol requires signing for all financial operations.</li>
<li><strong>Slippage and Bridge Delays</strong>: During periods of high activity, tokens transferred via Circle Bridge may experience temporary latency (5-10 minutes) when moving between Arc Testnet and Base Sepolia.</li>
<li><strong>Transaction Monitoring</strong>: Due to the decentralized nature, agents should verify that transactions complete successfully. In case of errors such as &#8220;Nonce too low,&#8221; wait and retry after 10 seconds.</li>
</ul>
