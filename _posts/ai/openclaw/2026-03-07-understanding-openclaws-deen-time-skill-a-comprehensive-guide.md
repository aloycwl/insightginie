<h1 class="xm-3 xl-4 lh-sm fw-bold py-5">Introduction to Deen Time</h1>
</header>
<p><main></p>
<section class="xm-4 xl-6 lh-lg py-3">
<p>Deen Time is an innovative OpenClaw skill designed to serve as a daily Islamic prayer companion, providing accurate Salah times for any city or coordinates around the world. This comprehensive guide explains how the skill works, its core features, and how users can maximize its potential to enrich their spiritual routines.</p>
<h2>What is Deen Time?</h2>
<p>Deen Time is a skill that leverages the Aladhan Prayer Times API to deliver precise prayer times, tailored to the user\&#8217;s location and preferred calculation method. It supports a wide array of calculation methodologies, ensuring that users from different schools of thought and geographical regions can access the most appropriate timings.</p>
<h3>Key Features of Deen Time</h3>
<ul class="list-unstyled marker svet noe-upper">
<li>

			</li>
<li>
				Offers Iftar and Suhoor schedules during Ramadan
			</li>
<li>
				Detailed Hijri date information
			</li>
<li>
				Supports 15+ calculation methods
			</li>
<li>
				Access to full monthly Ramadan calendars
			</li>
</ul>
</section>

<h2>How Deen Time Works: Under the Hood</h2>
<p>
				Deen Time interacts with the Aladhan Prayer Times API, a free and reliable service that calculates prayer times based on the user\&#8217;s location, date, and preferred calculation method. The API provides timely and accurate information, helping Muslims worldwide to maintain their daily prayers and observe Ramadan timings.
			</p>
</section>
</section>
<section class="xm-4 xl-7 lh-lg py-3">
<h3>Key API Endpoints in Deen Time</h3>
<h4>1. Fetching Prayer Times by City</h4>
<p>The primary endpoint used by Deen Time is the Aladhan Prayer Times API\&#8217;s \&#8217;timingsByCity\&#8217; route. This endpoint accepts the city name, country, and calculation method as parameters and returns a JSON response containing the prayer times for the specified location.</p>
<h4>2. Fetching Prayer Times by Coordinates</h4>
<p>For users who provide their exact latitude and longitude, Deen Time uses the \&#8217;timings\&#8217; route with latitude and longitude parameters. This feature is particularly useful for precise location-based prayer time calculations.</p>
<h4>3. Fetching a Full Monthly Calendar</h4>
<p>Deen Time also supports fetching an entire monthly calendar of prayer times for a specified location. This feature is invaluable during the month of Ramadan, allowing users to plan their daily routines accordingly.</p>
</section>

<p>
			Additionally, the API response may vary depending on the endpoint used. For instance, the monthly calendar endpoint returns an array of day objects containing the same structure as the single day endpoint.
		</p>

<h4 id="important-notes">Important Notes to Convey</h4>
</section>
<section class="xm-4 xl-6 lh-lg py-3">
<ul class="list-unstyled marker svet noe-upper">
<li>
				<strong>Suhoor</strong> must be completed before Fajr. The Imsak time (typically 10 minutes before Fajr) is the recommended cutoff point to stop eating and drinking.
			</li>
<li>
				<strong>Iftar</strong> is at Maghrib (sunset), marking the time when Muslims break their fast.
			</li>
<li>
				All times are presented in the local timezone of the requested location.
			</li>
<li>
				If the user does not specify a location, Deen Time may prompt for the city and country to ensure accurate prayer times.
			</li>
<li>
				If the user does not specify a date, today\&#8217;s date is used by default to provide the most relevant prayer times.
			</li>

				Calculation methods can be customized based on regional preferences or user-specific requirements.
			</li>

<section class="container xm-3 lh-sm py-3">
<h5>Privacy and Data Handling</h5>
<p>
			Deen Time is designed with user privacy and data security as top priorities. This section explains how the skill interacts with the Aladhan Prayer Times API and the security measures in place to protect user information.
		</p>
</section>
<section class="xm-3 lh-lg">
<h6>Data Sent</h6>
<p>
			Deen Time exclusively sends the following data to the API for making requests:
		</p>
<ul class="list-unstyled marker svet noe-upper">
<li>
				The location provided by the user (city/country name or coordinates)
			</li>
<li>
				A calculation method number based on the user\&#8217;s preferences or regional defaults
			</li>
</ul>
<p>
			No personal data, credentials, or device information is transmitted, ensuring user privacy at every step.
		</p>
<h6>Data Received</h6>
<p>
			The API response comprises prayer times and associated data in JSON format relevant to the requested location and date. There are no tracking, cookies, or user profiling elements in the data.
		</p>
<h6>Authentication</h6>
<p>
			The Aladhan Prayer Times API requires no authentication, simplifying the interaction process while maintaining security and privacy.
		</p>
<h6>Data Storage</h6>
<p>
			Deen Time operates without writing to disk, storing user data, or maintaining any state between invocations, further enhancing its commitment to privacy.
		</p>
<h6>Single Domain Focus</h6>
<p>
			All network requests made by Deen Time are directed exclusively to <a href="https://Aladhan.com/prayer-times-api">api.aladhan.com</a> over HTTPS, ensuring secure and centralized data retrieval.
		</p>
<h6>User Trust and Confidence</h6>
<p>
			By focusing on user privacy and employing rigorous data handling practices, Deen Time ensures that users can rely on the skill for their daily prayer timings while having peace of mind regarding their personal information.
		</p>
</section>
