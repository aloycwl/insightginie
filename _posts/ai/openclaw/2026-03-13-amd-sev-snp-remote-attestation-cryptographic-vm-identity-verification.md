<h2>What is AMD SEV-SNP Remote Attestation?</h2>
<p>AMD SEV-SNP (Secure Encrypted Virtualization &#8211; Secure Nested Paging) remote attestation is a cryptographic process that verifies the identity and integrity of a virtual machine running in a genuine AMD SEV-SNP confidential computing environment. This skill provides a complete workflow for proving that your VM is authentic and hasn&#8217;t been tampered with.</p>
<h2>When to Use SEV-SNP Attestation</h2>
<p>You should use this skill when:</p>
<ul>
<li>Proving to remote parties that your VM is running in a genuine AMD SEV-SNP environment</li>
<li>Verifying VM integrity before trusting it with sensitive secrets or cryptographic keys</li>
<li>Checking if SEV-SNP is properly configured and available on your system</li>
<li>Generating attestation reports for third-party verification</li>
<li>Validating the complete AMD certificate chain (ARK → ASK → VCEK)</li>
<li>Debugging attestation failures or certificate-related issues</li>
</ul>
<h2>The Complete Attestation Workflow</h2>
<p>The SEV-SNP attestation process follows a structured 6-step workflow that ensures comprehensive verification:</p>
<h3>Step 1: Detection &#8211; Is SEV-SNP Available?</h3>
<p>The first step checks whether the system supports SEV-SNP and if the necessary device files are accessible. This requires root access or membership in the &#8216;sev&#8217; group to access /dev/sev-guest.</p>
<h3>Step 2: Generate Report</h3>
<p>If SEV-SNP is available, the system generates a cryptographic attestation report. This report contains measurements of the VM&#8217;s current state, including firmware, hypervisor, and VM configuration.</p>
<h3>Step 3: Display Report Information</h3>
<p>The generated report is displayed with detailed information about the VM&#8217;s Trusted Computing Base (TCB) level, launch configuration, and other critical measurements.</p>
<h3>Step 4: Fetch AMD Certificates</h3>
<p>The system retrieves the AMD certificate chain from AMD&#8217;s Key Distribution Service (KDS) at https://kdsintf.amd.com. This includes the AMD Root Key (ARK), AMD Signing Key (ASK), and the VCEK certificate.</p>
<h3>Step 5: Verify Certificate Chain</h3>
<p>The certificate chain is validated to ensure it follows the proper hierarchy: ARK (self-signed) → ASK → VCEK. This confirms that the certificates are authentic and issued by AMD.</p>
<h3>Step 6: Verify Report Signature</h3>
<p>Finally, the attestation report&#8217;s signature is verified using the VCEK certificate. This confirms that the report was indeed generated by a genuine AMD SEV-SNP platform.</p>
<h2>Quick Start Commands</h2>
<p>Getting started with SEV-SNP attestation is straightforward:</p>
<pre><code>Check if SEV-SNP is Available
./scripts/detect-sev-snp.sh

Run Full Attestation
./scripts/full-attestation.sh [output_dir]
</code></pre>
<p>The full attestation script runs all six steps and outputs either &#8220;PASSED&#8221; or &#8220;FAILED&#8221; based on the verification results.</p>
<h2>Individual Step Scripts</h2>

</code></pre>

<h2>Technical Details</h2>
<p>Understanding the technical aspects helps with troubleshooting and implementation:</p>
<ul>
<li>AMD KDS URL: https://kdsintf.amd.com</li>
<li>Certificate Chain: ARK (self-signed) → ASK → VCEK</li>
<li>Report Signature: ECDSA P-384</li>
<li>Device: /dev/sev-guest (requires root or sev group membership)</li>
</ul>
