<h1>Unraveling OpenClaw&#8217;s Vibe-Check Skill: Audit Code for AI-Generated Patterns</h1>
<p>In the realm of software development, ensuring code quality and detecting AI-generated patterns is crucial. OpenClaw&#8217;s Vibe-Check skill is a powerful tool designed to audit code for &#8216;vibe coding sins,&#8217; which are patterns indicative of AI-generated code that may have been accepted without adequate human review. This article delves into the functionalities, features, and usage of the Vibe-Check skill, highlighting how it can enhance your code review process.</p>
<h2>Introduction to Vibe-Check Skill</h2>
<p>The Vibe-Check skill is an integral part of the <a href='https://github.com/openclaw/skills'>OpenClaw skills repository</a>. It is a specialized tool that focuses on detecting &#8216;vibe coding&#8217;—patterns in code that suggest it was generated by AI without sufficient human oversight. This skill provides a detailed report card, complete with scores and fix suggestions, to help developers improve their code quality.</p>
<h2>Key Features of Vibe-Check</h2>
<ul>
<li><strong>Comprehensive Audit:</strong> The Vibe-Check skill audits code to identify &#8216;vibe coding sins&#8217; and produces a scored report card. This report includes specific findings and suggestions for fixes, making it easier for developers to address issues.</li>
<li><strong>Versatile Targeting:</strong> Users can specify different targets for the audit, such as single files, directories, or Git diffs. This flexibility allows for a thorough review of code changes and existing codebases.</li>
<li><strong>Fix Suggestions:</strong> One of the standout features of the Vibe-Check skill is its ability to suggest fixes for identified issues. This is particularly useful for developers looking to quickly address potential problems in their code.</li>
<li><strong>Multi-Language Support:</strong> The skill currently supports Python, TypeScript, and JavaScript, making it versatile for a range of development projects.</li>
<li><strong>Engaging Report Format:</strong> The Vibe-Check skill generates a Markdown report that is designed to be engaging and easy to read. This makes it suitable for presenting to teams or stakeholders.</li>
</ul>

<h3>Using Vibe-Check</h3>
<p>Once installed, you can use the Vibe-Check skill to audit your code. Here are some common use cases:</p>
<h4>1. Auditing a Single File</h4>
<p>To audit a single file, use the following command:</p>
<pre>
vibe-check FILE
</pre>
<p>Replace `FILE` with the path to the file you want to audit.</p>
<h4>2. Auditing a Directory</h4>
<p>To audit an entire directory, use the following command:</p>
<pre>
vibe-check DIR
</pre>
<p>Replace `DIR` with the path to the directory you want to audit.</p>
<h4>3. Auditing Git Diffs</h4>
<p>To audit the changes in the last three commits, use the following command:</p>
<pre>
vibe-check --diff HEAD~3
</pre>
<p>This command focuses on the changes introduced in the specified commits, providing a targeted audit of recent modifications.</p>
<h4>4. Generating Fix Suggestions</h4>
<p>To generate a report with fix suggestions, use the following command:</p>
<pre>
vibe-check --fix DIR
</pre>
<p>Replace `DIR` with the path to the directory you want to audit. This command provides detailed suggestions for fixing identified issues.</p>
